# 싱크대 (Sink) 설계 규칙

> categoryId: `sink` / `island`

## 1. 기본 치수

| 항목 | 싱크대 | 아일랜드 |
|------|--------|---------|
| 기본 깊이 | 650mm | 800mm |
| 상부장 높이 | 720mm | - |
| 하부장 높이 | 870mm | 870mm |
| 상몰딩 높이 | 60mm | - |
| 다리발 높이 | 150mm | 150mm |
| 상부장 깊이 (BOM) | 295mm (고정) | - |
| 하부장 깊이 (BOM) | 550mm (고정) | - |

## 2. 레이아웃 형태

| 형태 | 설명 |
|------|------|
| I형 | 일자형 (기본) |
| L형 | ㄱ자형 (코너1 마감 필요) |
| U형 | ㄷ자형 (코너1 + 코너2 마감 필요) |

## 3. 워크스페이스 구조

- **상부장** (`pos: 'upper'`): 후드 위치 제외, 균등분배
- **하부장** (`pos: 'lower'`): 싱크볼, 쿡탑, 서랍장, 일반장, 키큰장(TL) 배치
- 상부/하부 각각 독립적 유효공간 관리

## 4. 모듈 타입

### 상부장
| 타입 | 설명 |
|------|------|
| storage | 일반 수납장 |
| hood | 후드 영역 (도어 없음) |

### 하부장
| 타입 | 설명 | 특수 속성 |
|------|------|-----------|
| storage | 일반 수납장 | isDrawer, isEL 토글 가능 |
| sink | 싱크볼장 | 기준 모듈 (isBase) |
| cook | 쿡탑장 | 기준 모듈 (isBase) |
| tall | 키큰장(TL) | doorCount, elCount 설정 |

### 모듈 속성
- `isFixed`: 고정 모듈 (자동계산 시 크기 유지)
- `isBase`: 기준 모듈 (싱크/쿡탑 — 자동배치 시 기준점)
- `isDrawer`: 서랍장 여부
- `isEL`: EL장(전동수납) 여부 — 활성화 시 너비 600mm로 고정 + isFixed

## 5. 키큰장(TL) 추가 규칙

```
기본 높이 = 전체높이(H) - 상몰딩(60) - 좌대(60)
기본 너비 = 600mm
기본 깊이 = 550mm
```

## 6. 자동계산 로직

싱크대는 `distributeModules()` 공통 알고리즘 사용 (→ common.md 참조)
- 상부장/하부장 각각 유효공간 기준으로 독립 분배
- 고정 모듈(isFixed) 제외 후 잔여공간을 비고정 모듈에 분배

## 7. 설비 항목

| 항목 | 기본값 |
|------|--------|
| 싱크볼 | 사각볼 850 |
| 수전 | 거위목 수전 |
| 후드 | 히든 후드 |
| 쿡탑 | 인덕션 |
| 식세기 | None / BuiltIn / FreeStanding |

식세기 = BuiltIn/FreeStanding 시 → `sinkLegHeight = 120`

## 8. 도어 표시 규칙

- `showDoors` 토글로 도어 오버레이 표시
- 도어 색상: `getDoorColor(specs.doorColorUpper)`
- 도어 간격: 3mm (스케일 반영)
